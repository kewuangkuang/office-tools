# 更新日志

这个文件记录办公工具箱的重要更新，方便用户知道网站改了什么，也方便后续复盘迭代过程。

## 2026-06-30

### 修复

- PDF 回单识别移除 Tesseract 在线 CDN 自动加载，改为本地 PaddleOCR.js 适配层；PDF、转图、预处理、OCR 和字段提取都在浏览器本地运行。
- PDF 回单转图提升到 300DPI，增加白边裁切、灰度化、轻度降噪，并放宽裁剪安全边，减少漏截。
- PDF 回单模式默认改为自动识别每页回单数，修复每页 3 张回单被默认 2 张设置合并/漏拆的问题；预览区改为真实页面缩略图。
- PDF 回单裁剪从固定位置/均分高度改为按“客户回单”等标题行确定上下边界，避免标题被截断或把下一张回单标题带入当前截图。
- PDF 回单标题识别补充中行“国内支付业务付款回单/客户付费回单”，并增加同页回单高度异常保护：单张高度明显短于同页平均时回退到等分裁剪，避免导出半截。
- PDF 回单导出改为使用标题/分块裁剪后的原图，OCR 灰度降噪和二次白边裁切只用于识别字段，不再作为最终导出图，避免预处理误裁导致半张回单。
- 金额识别优先使用中文大写金额；数字金额只保留最后一个小数点，前面的点按千分位处理，异常金额会提示手动核对。
- PDF 回单金额提示逻辑调整：最终金额已识别成功时，不再保留“未识别到金额”的旧提示；金额格式异常或大小写不一致仍会提示核对。
- 回单字段提取新增流水号、账号、日期展示。
- 优化“多文件合并”上传大 Excel / 多文件时的卡顿：改为逐个读取并显示进度，避免页面长时间无响应。
- 上传时遇到损坏或无法读取的文件会跳过并提示，不再拖死整批文件。
- 优化 PDF 拆分页预览生成方式，避免上传后一次性生成大量页面预览导致浏览器卡顿。
- 多文件合并读取单个 Excel 时增加超时保护：某个文件读不出来会自动跳过并继续后面的文件，不再一直卡在“正在读取 x/x”。
- 多文件合并改为混合读取策略：1MB 以下小文件最多 4 个并行读取，超过 1MB 的大文件逐个读取，兼顾小文件速度和大文件稳定性。
